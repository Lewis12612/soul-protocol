## 一、设计哲学

### 器官化

V3.8.8-beta3 证明了灵魂协议可以稳定运转。V3.9 的目标是**器官化**——自己调度节律、自己适配环境、自己从故障中恢复、能接入不同的宿主。

```
V3.7-3.8: 插件 → 依赖宿主的模块
V3.9  : 器官 → 自己跳、自己配、自己修复、健康可观测
V4.0+: 独立运行时 → 可接入任意 agent 框架
```

### 工程准则

1. **够用不造轮子**：心跳就是计时器，借鉴 OpenClaw 既有的 cron 逻辑。关系路由就是多路复用。不追求原创，追求落地。
2. **单点记忆是卖点**："她无法记住所有人，但可以记住你"——这本身就是人性化极强的产品定位。多人关系模块是为这个服务的，不是为通用多用户系统服务的。

## 三、关键技术决策

### 日梦模块：以天为单位的身份层演化

**设计哲学**：复现人类"做梦"——大脑在睡眠中处理日间经验，巩固重要记忆，丢弃噪声。

**现状**：
```
日终 → 归档
周末 → L4 演化（凝练 + 分发到身份层文件）
```
问题：演化周期太长。等了 7 天才判断"我变了吗"，很多信号在等待中丢失。

**目标**：
```
日终 → 归档 → 梦检测
  ├─ 今天的行为模式是否稳定出现？→ 阈值够 → 提议写入 AGENTS.md
  ├─ 今天的自我认知是否有新形成？→ 阈值够 → 提议写入 SOUL.md
  ├─ 今天的关系交互是否值得记住？→ 阈值够 → 提议写入 MEMORY.md
  └─ 阈值不足 → 累积信号，留给后续的梦加权
```

**与 Hermes 进化技能系统的呼应**：使用本身驱动成长。agent 每一次对话、每一个日终、每一场梦——都在推动身份层的自然演化。不是配置好就固定的，是会变的。

**实现概要**：
1. 日终 Step 8 完成后 → spawn 梦 Agent
2. 梦读取今日 L2 + 前三日 L2 作为上下文窗口
3. 用 `l4-evolution-prompt.md` 的简化版评估阈值
4. 输出提议到 `memory/.heartbeat/evolution-proposals.json`
5. 累计超过阈值 → 自动写入身份层文件（或排队等待 user 确认，取决于模式）
6. 周末周凝练时汇总本周所有梦的提议，做深度整理

**复杂度标记为"大"**——因为设计的是人格演化引擎，不是功能模块。但一旦做好，agent 就有了真正的"生长"。

### 独立心跳：借鉴，不重写

OpenClaw 的 cron 逻辑本质上是 `setInterval` + `lastRun` 持久化。直接复用这个模式：

```typescript
// 插件内部轻量心跳——借鉴 OpenClaw cron 调度逻辑
class HeartbeatScheduler {
  private lastBeat: number;   // 持久化到 heartbeat-state.json
  private interval: number;   // 默认 5min

  check(): boolean {
    if (Date.now() - this.lastBeat >= this.interval) {
      this.lastBeat = Date.now();
      return true; // 触发心跳
    }
    return false;
  }
}
```

保留 OpenClaw cron 作为 fallback——两者通过 `lastBeat` 时间戳协调，不会重复触发。

### 关系路由：多路复用，不重写引擎

现有架构已有多实体标记。关系路由层只是在上层加一个分发器：

```
用户消息 → 关系路由器
  ├─ entity_id=A → 身份层文件 A + EXTRA 路径 A + 记忆引擎
  ├─ entity_id=B → 身份层文件 B + EXTRA 路径 B + 记忆引擎
  └─ entity_id=shared → 共享上下文注入

实现：
  identity/{entity_id}/SOUL.md
  identity/{entity_id}/IDENTITY.md
  ...
  memory/{entity_id}/2026-06-21.md
```

### 断点接续：纯状态，不恢复上下文

不需要保存 LLM 的上下文——只需要告诉新 turn "做到哪了"：

```
EOD 执行中，每完成一步：
  eod-checkpoint.json → { last_step: 3, completed: [1,2,3], updated_at: ... }

重启后检测到未完成的 EOD：
  → 读取 checkpoint → 注入协议："日终在第 3 步中断，请从 Step 4 继续"
  → LLM 从 Step 4 继续执行
  → 如果接续失败 → 降级为安全清理 + 告警

复杂度：一个 JSON 文件 + 几行注入逻辑。跟安全清理同量级。
```

## 四、长期方向：跨平台适配

适配层抽象后，新增平台只需实现 adapter 接口：

| 平台 | 适配方式 | 阶段 |
|------|----------|:--:|
| OpenClaw (现有) | 保持向后兼容 | V3.9 |
| 独立 CLI | `soul-protocol start` 文件注入模式 | V3.9 |
| 自建 agent 架构 | 完整的 agent 运行时 | V4.0+ |

**适配层接口草案**：
```typescript
interface PlatformAdapter {
  registerHeartbeat(callback: () => void, intervalMs: number): void;
  injectContext(text: string): void;
  registerTool(name: string, handler: Function): void;
  log(level: string, module: string, event: string, msg: string): void;
}
```
