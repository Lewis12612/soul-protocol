## 一、核心新增

### 1. 三角因子指令解析（intent-resolver）

**问题**: 用户主动说「做日终」「跑周终」时系统完全无响应——conversation 路径无协议触发通道。

**方案**: 在 conversation 路径的睡意兜底之前插入 `intent-resolver` 模块：

```
用户消息 → conversation 路径
  → L2 注入
  → intent-resolver（新增）
      keyword + state → action
      direct → 执行协议 → return
      inquire → 注入询问提示词 → 继续 sleepiness
      pass → 继续 sleepiness
  → 睡意兜底（不变）
```

**三角因子**:

```
score = 0.3 × keyword + 0.4 × llm_intent + 0.3 × state_relevance
```

| 因子 | 权重 | 计算方式 | 说明 |
|------|:----:|------|------|
| keyword | 0.3 | 钩子内正则匹配 | 0/1，精确 + 模糊模式 |
| llm_intent | 0.4 | LLM 的 tool 调用行为 | tool 调用 = 1，不调 = 0 |
| state_relevance | 0.3 | 钩子内状态检查 | 0-1，含递减曲线 |

**决策逻辑**:

| keyword | stateRelevance | action | 行为 |
|:-------:|:--------------:|--------|------|
| 1 | ≥ 0.5 | direct | 直接执行协议 |
| 1 | < 0.5 | inquire | 注入询问提示词 |
| 0 | ≥ 0.7 | inquire | 状态强烈，询问 LLM |
| 0 | < 0.7 | pass | 不动作 |

**关键词映射**（7 组）:

| 协议 | 精确关键词 | 模糊模式 |
|------|-----------|---------|
| full | 日终、做日终、归档、睡觉了 | 日终\|归档\|睡觉 |
| medium | 检查状态、中等检查、更新记忆 | 检查\|中等\|更新.*记忆 |
| weekly | 周终、周凝练、周总结 | 这周\|周(终\|总结\|凝练) |
| monthly | 月终、月凝练、月总结 | 这月\|月(终\|总结\|凝练) |
| yearly | 年终、年凝练、年总结 | 今年\|年(终\|总结\|凝练) |
| sleepiness_query | 睡意、困不困、状态、困了 | 睡意\|困 |
| create_l2 | 创建L2、新建日记 | 创建L2\|新建.*日记 |

**状态因子**:

| 协议 | state=1 条件 | 递减规则 |
|------|-------------|---------|
| full | 睡意≥0.5 OR ≥8h | <0.5 线性降，<2h → ≤0.3 |
| medium | ≥2h | <2h 线性降 |
| weekly/monthly/yearly | 周末/月末/年末 + 文件不存在 | 布尔开关 |
| sleepiness_query | 恒定 0.5 | - |
| create_l2 | L2 不存在 | 存在 → 0 |

### 2. execute_protocol 工具

**注册方式**: `api.registerTool("execute_protocol", ...)`，不标 optional（自动生效）。

**参数**:
```ts
{ protocol: "full"|"medium"|"weekly"|"monthly"|"yearly"|"sleepiness_query"|"create_l2",
  reason: string }
```

**执行流程**:
```
LLM 调 tool → tool.execute():
  1. 去重检查（30min 窗口）
  2. 执行对应 check 脚本 / 状态查询
  3. 更新 last-eod.json / last-medium.json
  4. 格式化摘要返回 LLM
```

**7 种协议处理**:

| protocol | 行为 | 状态更新 |
|----------|------|----------|
| full | execSync(check-full.sh) → 格式化摘要 | last-eod.json |
| medium | execSync(check-medium.sh) → 格式化摘要 | last-medium.json |
| weekly/monthly/yearly | 检查文件是否存在 → 返回模板路径 | - |
| sleepiness_query | 读 last-eod.json → 三因子加权评分 | - |
| create_l2 | 检查 L2 是否存在 → 返回状态 | - |

### 3. L2 协议标记检测

**触发**: `before-prompt-build` 钩子中，eod-pending 消费之后。

**格式**: `<!-- PROTOCOL:full -->`（LLM 在回复中嵌入）

**检测**: 逆序遍历 `event.messages`，匹配最后一条 assistant 消息中的标记。

**去重**: 若本轮已有 `execute_protocol` tool 调用，跳过标记检测。

**标记移除**: 标记从消息内容中删除，用户不可见。

**协议构建**: `buildMarkerProtocol()` 读取已有状态文件构建轻量协议文本，不重复执行 check 脚本。

### 4. 工作流参考文档

**文件**: `references/protocol-workflow.md`（178 行）

**内容**: 协议概览表 / execute_protocol 参数 / 自然语言触发识别 / 依赖关系 / 睡意驱动 / 注意事项 / 场景速查

**注入**: startup 场景时注入一次，供 LLM 建立长期认知。

## 二、conflict 路径改造

### conversation 路径完整流程

```
用户消息 → before-prompt-build
  ├─ 0. eod-pending 消费
  ├─ 0.5 L2 标记检测
  ├─ 1. 场景分发 → conversation
  │     ├─ L2 记忆注入
  │     ├─ intent-resolver（三角因子）
  │     │     direct → executeProtocolDirect → updateLastEodTime → RETURN
  │     │     inquire → 注入询问提示词 → 继续
  │     │     pass → 继续
  │     └─ 睡意兜底（不变）
  │           exhausted/dreaming → EOD 协议
  │           sleepy → 提醒
  └─ 合并 prependSystemContext
```

### 去重矩阵

| 触发方式 | 去重机制 |
|----------|----------|
| intent-resolver direct | updateLastEodTime → sleepiness 和 watchdog 归零 |
| execute_protocol tool | 30min 冷却窗口 |
| L2 标记 | 同轮 tool 调用时跳过 |
| sleepiness 兜底 | 与 intent-resolver 互斥（direct 后已 return） |
| watchdog eod-pending | 消费后 updateLastEodTime |
| 多通道同时触发 | 30min last-eod 窗口统一拦截 |
