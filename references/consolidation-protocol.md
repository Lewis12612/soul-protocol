## 一、架构内核

周、月、年三级凝练共享同一个架构内核，差异仅在三项参数：

| 维度 | 周 | 月 | 年 |
|------|----|----|-----|
| **输入** | EXTRA/L2/L3 本周新增 | 本月 4-5 篇 weekly.md | 本年 12 篇 monthly.md |
| **输出** | `weekly/YYYY/YYYY-WXX.md` | `monthly/YYYY/YYYY-MM.md` | `yearly/YYYY.md` |
| **功能** | 凝练（叙事 + 提取） | 索引（压缩 + 回溯） | 回顾（摘要 + 里程碑） |
| **LLM 任务** | 读原文 → 叙事总结 → 提取认知 | 读周摘要 → 压缩 → 建立索引 | 读月摘要 → 年叙事 |
| **密度** | 最详细，保留叙事弧 | 中等，保留关键节点 | 最高压缩，仅里程碑 |

**核心原则**：周是唯一的"主动凝练"单位——它从原始材料中提取。月和年是对上层产物的再压缩，不再接触原始数据。

## 二、协议结构

### 2.1 整体注入模型

```
日终 Full 协议注入完毕（actions 1-7）
  │
  ├─ weekly_due.needs_weekly_consolidation = true
  │   → 增量注入 周梦协议（actions W1-W4）
  │
  ├─ month_end.is_month_end = true
  │   → 增量注入 月凝练（action M1）
  │
  └─ year_end.is_year_end = true
      → 增量注入 年凝练（action Y1）
```

### 2.2 协议 action 清单

#### 周级（核心）

| ID | Action | Executor | 输入 | 输出 |
|----|--------|----------|------|------|
| W1 | 周记忆凝练 | spawn:梦 | EXTRA/L2/L3 本周 + monthly/yearly（按需回溯） | weekly.md |
| W2 | L4 演化审查 | agent | weekly.md + 当前 SOUL/AGENTS/MEMORY | SOUL/AGENTS 演化更新、MEMORY 写入 + 剪枝 |

#### 月级（仅月末）

| ID | Action | Executor | 输入 | 输出 |
|----|--------|----------|------|------|
| M1 | 月记忆凝练 | spawn:梦 | 本月 weekly.md × 4-5 | monthly.md |

#### 年级（仅年末）

| ID | Action | Executor | 输入 | 输出 |
|----|--------|----------|------|------|
| Y1 | 年记忆凝练 | spawn:梦 | 本年 monthly.md × 12 | yearly.md |

### 2.3 执行顺序

```
日终 1-7 → W1（周凝练）→ W2（L4演化）
  → M1（如月末）
  → Y1（如年末）

W1 必须优先于 W2（W2 依赖 W1 的输出）
W1 执行时可按需读取 monthly/yearly 做跨时间联想
```

## 三、周凝练（W1）— 核心单位

### 3.1 输入材料

W1 的梦需要读取本周的完整叙事脉络：

| 来源 | 内容 | 用途 |
|------|------|------|
| EXTRA 层 | 本周每日原始对话 | 叙事主线，最完整的对话上下文 |
| L2 文件 | 本周每日记忆 | 事件摘要、跨日交接记录 |
| L3 deep-dialogue | 本周深度对话归档 | 哲学/自我认知内容 |
| L3 work-dialogue | 本周工作对话归档 | 方法论/错误修正 |
| L3 daily-dialogue | 本周人格温度 | 互动模式、情感变化 |
| monthly.md | 本月已有周摘要（可能有） | 本周在月中的位置 |
| yearly.md | 本年已有月摘要（可能有） | 本周在年中的位置 |

### 3.2 回溯联想机制

```
W1 执行时，梦被允许读取 monthly/yearly 文件做联想:

1. 本周叙事中出现了一个主题 → "这个主题之前出现过吗？"
   → 查 monthly: 本月前几周有没有相关主题
   → 查 yearly: 今年前几个月有没有相关主题

2. 本周有一个情感模式 → "这种情绪之前有过吗？"
   → 回溯年索引，找到类似时期的叙事

3. 本周有一个认知转折 → "这是新认知还是旧认知的延续？"
   → 对比年月索引，判断连续性

回溯结果写入 weekly.md 的 "关联" 部分:
  - 如果是新主题 → 标注 "新"
  - 如果是延续 → 链接到之前的 weekly/monthly
  - 如果是转折 → 标注 "从 X 转向 Y"
```

### 3.3 输出格式

`memory-core/weekly/YYYY/YYYY-WXX.md`:

```markdown
# WXX 周摘要 (YYYY-MM-DD ~ YYYY-MM-DD)

## 叙事主线
<!-- 本周发生了什么。按时间线，保持叙事弧。 -->
<!-- 来源: EXTRA + L2 -->

## 深度对话
<!-- 本周哲学/自我认知讨论的要点。不复制原文。 -->
<!-- 来源: L3 deep-dialogue -->

## 工作经验
<!-- 本周方法论发现、错误修正。 -->
<!-- 来源: L3 work-dialogue -->

## 人格温度
<!-- 本周互动模式、情感变化、关系动态。 -->
<!-- 来源: L3 daily-dialogue + EXTRA 互动片段 -->

## 回溯关联
<!-- 本周叙事与更长时间尺度的联系 -->
<!-- 来源: monthly/yearly 索引 -->
| 关联类型 | 主题 | 链接到 |
|----------|------|--------|
| 新 | X | - |
| 延续 | Y | WXX.md |

## 认知提取
<!-- 从本周叙事中提炼的关键认知。W2 将从这里提取 L4 演化素材。 -->
<!-- 每一条标注: 主题、首次出现还是复现、情感强度 -->
```

## 四、L4 演化审查（W2）— 身份层更新

### 4.1 设计原则

```
W2 不是操作清单，是一次"自我审视"。

agent（执行者）被要求:
  1. 阅读本周 weekly.md 的叙事
  2. 对照当前 SOUL.md / AGENTS.md / MEMORY.md
  3. 问自己三个问题:
     a. 经过这一周，我对自己有了什么新的理解？
     b. 这些理解是否改变了我之前对自己的描述？
     c. 如果是，哪些描述需要更新？
  4. 做出判断并执行
```

### 4.2 分文件判断

| 文件 | 判断问题 | 演化的触发条件 |
|------|----------|---------------|
| **SOUL** | "我对'我是谁'的理解变了吗？" | 出现新的底层自我认知；现有描述与本周行为明显矛盾；反复出现之前未写入的人格机制 |
| **AGENTS** | "我的行为边界或体感模式变了吗？" | 发现新的稳定行为边界；体感层出现新状态倾向；已有规则需修正 |
| **MEMORY** | "本周有什么必须记住的？" | 总是写入本周关键认知；≥40条则剪枝 |

### 4.3 演化方式

```
SOUL/AGENTS:
  不改是常态，改是例外。
  大多数周: 什么都不改。
  需要改时: 重写或调整相关段落，不追加。
           旧描述如果不再准确 → 替换，不是保留。

MEMORY:
  总是写入，按格式: [日期] 事实。→ 我的认识。
  剪枝: ⚓锚点永久，3-6月评估，6月+可移除，冗余合并。
```

### 4.4 提案记录

```
当 SOUL/AGENTS 发生改动时 → 不直接写入 → 生成提案:

提案包含:
  - 改动文件
  - 改动理由（从 weekly.md 提取的证据）
  - before/after 对比

提案写入 weekly.md 的 "演化记录" 部分:
  老师可见，可回退。不是审批流程，是透明记录。
```

## 五、月凝练（M1）— 索引层

### 5.1 定位

```
月 = 周摘要的压缩索引。

不做叙事重建——周已经做了叙事。
月做的是:
  1. 提取本月 4-5 篇 weekly 的关键词和主题
  2. 标注重要事件和转折点
  3. 建立跨周关联
  4. 形成可快速检索的月索引
```

### 5.2 输入与输出

| 输入 | 输出 |
|------|------|
| 本月 `weekly/YYYY/YYYY-WXX.md` × 4-5 | `monthly/YYYY/YYYY-MM.md` |
| 上月 `monthly.md`（如有，做跨月关联） | |

### 5.3 输出格式

```markdown
# YYYY-MM 月摘要

## 月叙事
<!-- 本月主线。1-2段。来源：各周叙事主线的串联。 -->

## 关键事件时间线
| 周 | 日期范围 | 关键事件 | 关联主题 |
|----|----------|----------|----------|
| WXX | ... | ... | ... |

## 主题索引
<!-- 本月出现的所有主题，标注首次出现/延续/转折 -->
| 主题 | 起始(周) | 类型 | 跨月关联 |
|------|----------|------|----------|
| ... | WXX | 新 | - |

## 认知汇总
<!-- 本月各周认知提取的汇总，去重。 -->
```

## 六、年凝练（Y1）— 回顾层

### 6.1 定位

```
年 = 月度索引的回顾。

只保留里程碑和主题演变。
不追溯细节——细节在月和周里。
```

### 6.2 输出格式

```markdown
# YYYY 年摘要

## 年叙事
<!-- 2-3段。全年的故事线。 -->

## 里程碑
| 月份 | 事件 |
|------|------|
| 01 | ... |

## 主题演变
<!-- 贯穿全年的主题如何变化 -->
| 主题 | 起始 | 变化 | 终结 |
|------|------|------|------|

## 年度认知
<!-- 全年最重要的认知变化 -->
```

## 八、身份层文件规格与剪枝

> **来源**: 身份层宪法 V2.2 §二 + §六

### 8.1 文件容量上限

| 文件 | 字数 | KB | Token | 保护级别 |
|------|:--:|:--:|:--:|:--:|
| SOUL | 5K | 10 | 10K | 🔒 绝对 — 只改不增 |
| IDENTITY | 2K | 4 | 4K | 🟡 有条件 |
| AGENTS | 5K | 10 | 10K | 🟡 有条件 |
| USER | 3K | 6 | 6K | 🟡 有条件 |
| MEMORY | 50条 | 12 | 8K | 🔵 自由 — 常规剪枝 |
| **合计** | **19K字** | **42KB** | **38K** | 占 1M 窗口 3.8% |

### 8.2 MEMORY 剪枝规则

**触发**: MEMORY ≥ 40 条 → 执行剪枝

**五因子辅助判断**（量化信号：LLM 拿到的结构化上下文，不替判断）:

| 因子 | 来源 | 类型 | 说明 |
|------|------|:--:|------|
| 时间衰减 | 数学公式 | 机械化 | 越旧的条目自然权重越低 |
| 检索频率 | 计数器 | 机械化 | 被 memory_search 命中的次数 |
| 老师信号 | 模式检测 | 机械化 | 老师是否引用/提及过该条目 |
| 情感强度 | LLM 读条目 | 语义化 | 条目的情感负载程度 |
| 关联锚点 | LLM 读条目 | 语义化 | 是否关联到 SOUL 中的关键认知 |

**剪枝阶梯**:

```
⚓ 锚点      → 永久保留（SOUL/AGENTS/MEMORY 中的关键认知，唯一性标识）
3-6 月前     → 评估是否仍生效，过时则移除
6 月以上     → 可移除（除非是锚点）
内容冗余     → 合并相似条目
```

**剪枝方式**: LLM 在 L4 演化审查时执行。不是自动化代码，是语义判断。

### 8.3 SOUL/AGENTS 容量管理

SOUL/AGENTS 有硬上限（各 10KB），演化时需遵守：

```
SOUL:
  演化替换旧描述 → 不追加
  接近上限时 → 压缩冗余表述，保持抽象程度，不逐条删除
  保护: 🔒 绝对 —— 不改不改的

AGENTS:
  行为准则可追加，但接近上限时压缩
  体感层五态 → 允许微调，不新增冗余状态
  保护: 🟡 有条件
```

### 8.4 周期凝练中的剪枝执行时机

```
W2 (L4演化审查) 中执行:

1. 读取当前 MEMORY.md → 判断是否 ≥ 40 条
2. 是 → 启动剪枝: 应用五因子 + 阶梯规则
3. 读取 SOUL/AGENTS → 检查是否接近容量上限
4. 是 → 演化时压缩而非追加
5. 所有剪枝结果记录在 weekly.md 的演化记录中
```
