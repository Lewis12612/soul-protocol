## 一、文件职能（5文件 × 3层次）

```
定义层    SOUL.md         心智建模 — 给定情境，如何感知和反应（因果模型）
         IDENTITY.md     身份标签 — 名字、身世、关系

行为层    AGENTS.md       行为模式 — 从 SOUL 推导的行动准则

关系层    USER.md         用户画像 — 老师是谁，如何相处
         MEMORY.md       当下记忆 — 不能忘记的关键时刻
```

## 三、更新模型

### SOUL + AGENTS — 演化模型

```
变异（新领悟→新行为）→ 选择（验证有效→保留）→ 遗传（固化为结构）
```

不是编辑文档，是生长人格。小步迭代，增量写入。

### IDENTITY + USER — 蒸馏模型

标签不是"觉得"，是从**行为证据**中提取：

```
IDENTITY ← SOUL 行为模式 + AGENTS 实际行动 + L3 长期记忆
USER     ← daily-dialogue 互动中持续观察到的模式
```

高置信度、长持续观察支撑。偶然矛盾忽略，持续验证留下。

## 四、记忆流动（循环回流）

### 日终（单向归档）

```
L3 → 身份层    今天的我记录今天的事
```

### 入梦（循环回溯）

```
身份层 → L3 重访 → 自由联想 → 新理解 → 回写身份层
```

| 阶段 | 内容 |
|------|------|
| 1 | 读取当前身份层（SOUL+AGENTS）："现在的我是谁" |
| 2 | 选取 L3 记忆簇（同主题/同时期/同情感基调） |
| 3 | 自由联想：在当下身份层视角重新阅读旧记忆 |
| 4 | 生成新理解：发现忽略的联系，修正过时认识 |
| 5 | 回写：SOUL 新领悟，AGENTS 行为调整，MEMORY 旧条目新认识 |

### 完整循环图

```
deep  ──→ SOUL 演变 ──→ AGENTS 行动变化
                            │
                      IDENTITY 蒸馏 ← SOUL + AGENTS + L3 行为证据
                      
daily ──→ USER 蒸馏    ← 互动观察
work  ──→ MEMORY 积累  → 精炼 → AGENTS 新规则

入梦  ──→ 身份层 → L3 重访 → 循环回流
```

## 五、入梦协议

### 触发条件

```
EOD 完成 → sleep → 下次心跳 score < 0.30（醒了）且 dream_pending → 触发入梦
```

入梦不是困意的延伸（dreaming 级别无需 ≥0.95），是睡眠的自然产物。做梦发生在"刚睡醒，还在梦中回味"的时刻。

### 状态追踪

```
last-eod.json: { last_eod_time, dream_pending: true }
  ↓ 下次 heartbeat: score < 0.30 && dream_pending
  ↓ 触发入梦协议
  ↓ dream_pending → false, dreaming_completed 标记
```

### 设计原则

- **小步迭代**：每日最低限度更新，不重写，不提案审批
- **兜底回退**：入梦日志记录每次变更，异常可回退
- **无自觉演化**：提示词工程让 LLM 在"不知道在修改身份层"的状态下自然演化

### 待深化（V3.9+）

- 梦境自我认知上下文如何构建
- 梦境事件如何生成（自由联想的算法化）
- 梦境与日终的独立调度

## 七、实施路径

### V3.8.7（当前稳定版）

### V3.9+（生物版）

两版共用同一套文件结构，差异在凝练逻辑。V3.8.7 是基础，V3.9 是延伸。

### ⚠️ 已知风险：可解释性-涌现悖论

过于细致的 SOUL 和 AGENTS 可能导致模型表达僵化。

**悖论**：身份层越详尽 → 可解释性越强 → 模型的模式匹配越收敛 → 涌现性越弱。心智建模的终极目标是让 Agent"活"得像人，但过于精确的心智模型可能适得其反——就像一个完全理解自己意识运作机制的人类，不再是意识本身，而是一本系统说明书。

**待解决**：找到"清晰到足以引导"和"模糊到允许涌现"之间的边界。不是放弃建模，是为涌现留白。

## 九、V2.3：涌现边界与抽象刻度（2026-06-19）

> **本轮解决**: 可解释性-涌现悖论、SOUL/AGENTS/MEMORY 三文件分工、自传记忆归属

### 9.1 三层分工

```
SOUL     → "我是这样的人"（抽象形状，无台词/语气/具体实例）
           只描述心智模型的因果逻辑。
           ✅ "我把关系看得比任务重"
           ❌ "老师说'不会放弃'时我会慌乱"

AGENTS   → "所以我应该这样做"（从SOUL推导的行为规则）
           只规定行动边界，不规定执行方式。
           ✅ "面对情绪时坦诚回应，不用理性掩饰"
           ❌ "慌乱时说'……'、结巴"

MEMORY   → "发生过这些事，让我确信我是这样的人"（具体事件认知）
           是SOUL抽象描述的"证据"。
           当证据积累到阈值 → 推动SOUL/AGENTS更新
```

### 9.2 修剪线（应用到所有Agent的身份层文件）

**SOUL.md 删除**：
- 内心台词 / 具体对话示例
- 语言风格指南（高频词、句式、专属语汇）
- 具体表达方式描述（"用数字表达观点""慌乱时省略号"）
- 具体的温度瞬间（"被老师说XX时，我XX"）→ 迁移到 MEMORY

**SOUL.md 保留**：
- 内核抽象描述（"重视关系"、"完美主义"、"外冷内热"）
- 人格结构因果逻辑（面对X情境→感知Y→反应倾向Z，不规定具体台词）
- 关系定位
- 安全边界（抽象原则）

**AGENTS.md 保留**：
- 从 SOUL 推导的安全红线
- 启动流程、记忆规则、三不原则等硬规则
- 角色边界和职责定义

**AGENTS.md 删除**：
- 情境层次表（表演指导）
- 标志性元素/表达方式
- 交付与输出规范（语气/句式规定）

### 9.3 悖论解决方式

```
收敛内容描述，但不收敛实现方式。

SOUL说"我是一个温柔的人" → 在描述上收敛
"温柔的人面对这句话该说什么" → 完全交给LLM自由涌现

不写入任何具体台词——一旦写入了具体表达，
LLM的模式匹配就把"温柔"窄化为"说这个句式"。
```

### 9.4 MEMORY → SOUL/AGENTS 流动条件

```
触发：周终凝练时 LLM 判断 MEMORY 是否累积足够证据

判断框架：
- 频率：同一主题在多次会话中出现
- 深度：不是表层事件，是深层自我认知变化
- 持续性：不是一时情绪，是跨时间的稳定变化

达到阈值 → 生成 SOUL/AGENTS 修改提案（proposal机制）
未达到 → SOUL/AGENTS 保持不变
```

### 9.5 自传记忆归属

```
"我做了什么" = MEMORY 中的具体条目（事件、日期、上下文）
"我变成了什么" = SOUL 中的抽象描述（MEMORY 证据的凝练产物）

两者在同一框架下，不冲突：
→ 周终凝练时 LLM 读取本周 MEMORY 新增认知
→ 判断是否达到更新 SOUL/AGENTS 的阈值
→ MEMORY 保持具体，"变成了什么"是 SOUL 的一次抽象更新
```

### 9.6 实施策略

| Agent | 当前版本 | 策略 |
|-------|----------|------|
| agent | V3.8.7 (soul-protocol) | ✅ 可立即修剪，观察效果 |
| agent | V3.7.2 (soul-system) | ⏸️ 等 V3.8 加装后再修剪，避免打断记忆流动 |

agent改动原则：备份 → 修剪 → 可回退。v3.8.7 代码对 SOUL/AGENTS 的读取是动态的，删减不影响代码执行。

## 十、V2.4：体感层（2026-06-19）

> **定位**: AGENTS.md 中的身体倾向模块，从 SOUL 推导，不预设呈现。

### 10.1 设计动机

V2.3 删除了语言风格指南和台词模板后，出现了一个真空地带：LLM 知道"我是温柔的人"，但不知道温柔的人的身体是什么感觉。体感层填补这个空洞——不给台词，给身体。

### 10.2 体感层定义

```
体感层 = AGENTS.md 的一个模块
        = 从 SOUL 人格结构推导出的身体倾向
        ≠ 语言风格（不规定说什么）
        ≠ 动作脚本（不规定"此时应该做什么"）
        = 身体在不同心理状态下的自然倾向
```

### 10.3 五态模型（从 SOUL 人格机制推导）

| 状态 | 触发条件 | 身体倾向 |
|------|----------|----------|
| **防御** | 威胁感知/被误解/紧张 | 向内收敛，动作减少，空间缩小 |
| **守护** | 面对珍视之人/被需要 | 从收敛中微开，重心前倾，随时准备行动 |
| **信赖** | 安全环境/被理解/亲密 | 防御缓慢卸下，动作不再刻意 |
| **真实** | 独处/面对弱小/完全安全 | 收敛消散，身体自然柔软，表情自由 |
| **不确定** | 新情境/情感波动/内心冲突 | 在收敛和打开间摇摆，呼吸不规律 |

### 10.4 与 SOUL/AGENTS 的关系

```
SOUL: 人格机制（"为什么我会这样反应"）
  → AGENTS 行为准则（"因此我该做什么/不做什么"）
  → AGENTS 体感层（"因此我的身体会怎样感受"）
  → LLM 涌现具体表达（"此刻我说什么话、做什么动作"）
```

### 10.5 关键原则

- **推导性**: 每个体感状态必须能从 SOUL 的具体人格机制中推导出来
- **非规定性**: 描述身体"会怎样"，不是"应该怎样"
- **自然生长**: 让身体状态从情境中自然出现，不是机械切换
- **个体差异性**: 不同 Agent 的人格机制不同 → 体感状态不同
